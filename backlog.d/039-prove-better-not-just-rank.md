# Make the foundry prove "better," not just rank it

Priority: P0 · Status: ready · Estimate: XL

## Goal
A Daedalus "win" is a statistically defensible claim that a candidate beats the baseline (and its rivals) on the task — confidence-bounded, consistency-aware, and robust to the search trajectory — not the top point-estimate of a small, noisy tournament.

## Oracle
- [x] Every certified candidate's report shows a 95% CI on (candidate − baseline) reward, computed with **clustered** standard errors over tasks (tasks from the same source repo form one cluster). _(child-1; repo-clustering activates when 040 lands source_repo labels — per-task today.)_
- [x] `improved_over` / certification accept a candidate only when the CI excludes 0 (or a configured minimum detectable effect), not merely when n ≥ certify_trials. _(child-2; gate placed on certification — `improved_over` left as the plateau signal, see child-2 note.)_
- [x] The report surfaces a per-candidate **consistency** metric (pass^k-style: fraction of trials at/above a reward floor), reported separately from mean reward. _(child-3.)_
- [x] A robustness check runs ≥2 independent seed trajectories and flags when their certified tops diverge by more than the pooled noise (basin-trap detector). _(child-4: `stats::basin_divergence` + `daedalus basin`; the live ≥2-seed run is spend-gated.)_
- [x] A power note: given measured per-task variance, the report states the minimum n to detect the target reward delta at 95% confidence. _(child-5; scales clusters.)_

## Verification System
- Claim: certified candidates are genuinely better — not noise, not a basin artifact.
- Falsifier: re-running a "certified win" at larger n, or from a second seed, overturns the ranking.
- Driver: `daedalus run` on pr-review-v2 with raised `--certify-trials` + a second `--rng-seed`; a Rust stats unit-battery over known synthetic reward distributions.
- Grader: CI/significance computed in Rust (bootstrap or paired t / Wilcoxon + clustered SE), asserted against known-distribution fixtures in `bin/gate`.
- Evidence packet: report.md CI table + consistency column + divergence flag + holdout-ledger row.
- Cadence: every `run`; the stats battery runs in `bin/gate`.

## Children
1. ✅ **Landed** — Clustered-SE 95% CIs on reward deltas in report.md + loop.json. **Why:** "Adding Error Bars to Evals" (Miller, arXiv 2411.00640) — evals are experiments; report ±1.96·SE and use clustered SE when items are correlated. pr-review-v2's 10 tasks come from 2 repos (rich, pygments) → naive SE understates variance. **Done:** `daedalus_core::stats` (CR1 cluster-robust SE; singleton clusters provably reduce to the standard SE of the mean), emitted per certified candidate to report.md (`## Reward delta vs baseline (95% CI)`), loop.json (`reward_delta_cis`), and pareto.json. Baseline = the `null` floor; `cluster_of` injects the cluster key, defaulting to per-task until [[040]] lands `source_repo` labels (then same-repo tasks collapse and the SE widens). Live evidence: on the real pr-review-correctness run, all 5 seed "wins" over null were insignificant (every 95% CI spanned 0) — the thesis made mechanical.
2. ✅ **Landed** — CI-gated certification (configurable MDE). **Why:** today acceptance compares point-estimate means against a within-run noise band (`search_loop.rs::trial_noise`); the foundry's own repro showed 0.69 vs 1.000 in-search variance (ROADMAP.md). **Done:** certification now requires `stats::partition_certified` to place a candidate in the certified set — its (candidate − null) CI lower bound must exceed `--min-effect` (default 0.0, rejected if negative). Trial-complete-but-insignificant candidates are reported separately; an all-underpowered run prints a loud "no provable win" banner. **Scope decision:** `improved_over` is deliberately left unchanged — it drives only the plateau/exploration stop and the history trail, not population membership or recommendation, so gating it on significance would shorten searches without adding rigor (the acceptance decision that matters is certification → recommendation, which is now gated). Live: real pr-review-correctness run certifies 0/5 at MDE 0.0 (all CIs span 0).
3. ✅ **Landed** — Per-candidate consistency / pass^k metric. **Why:** τ-bench (Sierra) + "Consistency as a Testable Property" (arXiv 2605.10516) — mean hides reliability; pass^k (succeed across ALL k) is the deployability axis. A reviewer right 60% of the time is not shippable at any mean. **Done:** `stats::{candidate_consistency, pass_hat_k, consistency_markdown}` — pass rate at a reward floor + pass^k (`C(c,k)/C(n,k)`), emitted as a `## Reliability` section in report.md and a `consistency` block in loop.json, separate from mean. `--consistency-floor` (default 1.0 = perfect-rate). Live (pr-review-correctness): the top candidate is "right" 57% of the time with pass^5 = 0.028 — not deployable, exactly as the mean hides.
4. ✅ **Landed (detector + CLI; live run spend-gated)** — Trajectory-divergence / basin-trap detector (≥2 seeds; compare certified tops at holdout). **Why:** the search is single-population reflective hill-climbing (`search_loop.rs`) with no basin escape; ~2× seed budget (~$0.60) buys a robustness signal (operator-lane finding). **Done:** `stats::basin_divergence` (+ `RunTop`/`BasinVerdict`) compares ≥2 seed runs' certified tops; flags a basin trap when seeds crown different compositions whose reward gap exceeds the pooled 95% noise band — different winners *within* noise are reported as equivalent optima, not a hard flag. `daedalus basin <run-dirs…>` reads each run's pareto.json and prints the verdict. Reduces each distinct composition to its best run (so within-composition trial noise can't masquerade as divergence), and distinguishes missing CI data (`se: None` → untestable) and missing composition identity (→ indeterminate) from a real trap — all critic-hardened. Offline-proven: synthetic trap/robust/equivalent/untestable/indeterminate/error paths all correct; two real pr-review-v0 runs are reported DIVERGENT (untestable) — different winners, but those pre-039 runs carry no CI to test the gap. **Spend-gated residual:** a rigorous live verdict needs ≥2 fresh searches with the new CI emission (~$0.60 via `--rng-seed`) — authorize to run.
5. ✅ **Landed** — Power/sample-size note from measured variance. **Why:** ROADMAP already concedes "contract-grade claims need n≥5" — make n a computed function of variance + target MDE, not a guess. **Done:** `DeltaCi::min_clusters_to_significance` — `G ≥ n_clusters·(1.96·se/point)²`, the number of independent **clusters** at which the *observed* effect's 95% CI is expected to just reach 0. Emitted as the `clstr→95%` column in the CI table + `min_clusters_95` in loop.json (rides the child-1 emission; no new wiring). **Critic correction:** scales the cluster count, not tasks — the CR1 SE shrinks with clusters (1/√G), so adding tasks inside existing repos does not help; once 040 labels repos, the lever is more repos. Marginal threshold (≈50% detection at exactly G); budget more for reliable detection. Live (pr-review-correctness, per-task clustering so G=n_tasks today): seed1's +0.2857 needs **23** clusters (has 7), seed4's +0.20 needs **68** (high variance), seed2's zero effect shows "—". n is now computed, not guessed.

## Notes
This is THE gap between "ranks candidates" (today) and "discovers provably high-quality configs" (the stated mission). It builds ON the existing rigor (oracle/null/probe gates, holdout ledger, seeded RNG) by adding the inferential layer. Cost: the statistics are offline/free; spend only rises from larger n and the second seed trajectory. Pairs with [[040]] — valid statistics over a valid arena is what makes discovery trustworthy. External grounding: arXiv 2411.00640, τ-bench/Sierra, arXiv 2605.10516.
