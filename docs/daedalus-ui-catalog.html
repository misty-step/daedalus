Make each task the root of the foundry.

Daedalus needs an operator interface where every benchmark, hypothesis, candidate composition, run, trace, and launch package hangs off a task contract. This pass keeps the strongest workbench views and pivots the information architecture around bespoke agent discovery.

Catalog verdict

Use the Console aesthetic as a Task Workbench, Matrix as the comparison core, Trace Lab as drill-down, and Gate Room as the launch boundary. Keep Loop Board for hypothesis planning.

Task-first model

The UI root is the task contract, not the lab.

A task can be single-agent, such as launch-contract review, or suite-shaped, such as PR review with specialist members and a master synthesizer. The interface keeps that distinction visible without forcing the operator into separate products.

Recommendation

Build a Task Dossier shell with Matrix, Trace, Hypothesis, Swarm, and Launch tabs.

  • Default screen: Task Dossier, because Daedalus is task-first and every downstream object needs the task thread.
  • Primary work tab: Experiment Matrix, because task-level score grids prevent mean-score overclaiming.
  • Deep inspection: Trace Lab, because every failed trial needs a path from score to artifact.
  • Launch handoff: Gate Room, because human approvals and plane boundaries are product truth.
  • Planning surface: Hypothesis Board, because task research needs visible hypotheses and postmortems.
  • Pre-G1 shaping: Idea Map, because raw reviewer-lane speculation belongs outside measured evidence.
