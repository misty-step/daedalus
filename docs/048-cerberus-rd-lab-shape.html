VERIFICATION SYSTEM

Claim: Threshold can compare Cerberus autonomous review substrates credibly enough to recommend what Cerberus should run next.

Falsifier: invalid artifacts pass, substrate failure is scored as model quality, unknown cost is guessed, or an inconclusive freeze still produces a recommendation.

Driver: Cerberus fixture/live runs, Threshold import/scoring/report, arena-freeze, arena-validate, bin/gate.

Evidence: request, artifact, transcript, run directory, report, freeze reports, critic receipt.

OPERATOR FLOW

1 choose task

Select arena, task family, objective, context tier, and whether this is fixture-only or live-spend.

Stop: no frozen arena or missing answer key.

2 choose matrix

Select OpenCode, OMP, and Pi/comparator arms with one controlled hypothesis per run.

Stop: model missing from refreshed primitives.

3 run

Execute fixture import first, then live Cerberus receipts only with explicit cost/time/security posture.

Stop: hidden files, secrets, or prompt/diff material leak into unsafe surfaces.

4 analyze

Read task grid, invalid-output reasons, context overclaims, false positives, misses, cost, and latency.

Stop: result moves only inside observed trial noise.

5 promote or postmortem

Emit a sandbox-only recommendation or plateau postmortem. Production gates remain human.

Stop: G2/G3/G4/G5 approval is missing.
