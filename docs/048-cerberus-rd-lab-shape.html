DaedalusCerberusR&D lab

Make Daedalus useful by proving the best autonomous review substrate for Cerberus first.

Chosen direction: build a Cerberus-focused upstream lab that compares OpenCode, OMP, and current Pi-style review candidates by validated artifacts, cost, latency, and lifecycle truth. Generalize only after this lab produces trustworthy evidence.

Contract

Outcome: Daedalus can recommend a Cerberus review configuration because it measured agent substrates, not because a report favored one.

Proof: same review request, same arena, comparable run records, strict artifact validation, no production posting.

Stop if: Cerberus artifact validation cannot be mapped into Daedalus scoring, OpenCode output cannot be captured deterministically, or the freeze probe remains inconclusive for the chosen arena.

Current State

Daedalus V1 measures pi candidates plus reference oneshot/null/oracle. Cerberus now has opencode, omp, and fixture harnesses around ReviewArtifact.v1. Bitterblossom and Olympus are consumers/control planes, not the lab.

Thesis

Supervised agents can be terminal-first. Unsupervised agents need server/session shape, strict artifacts, durable lifecycle, bounded env, and replayable receipts. OpenCode is the first autonomous substrate candidate to test, not a belief to install globally.

First Deliverable

A Cerberus substrate-eval vertical slice: Daedalus runs or ingests Cerberus OpenCode and OMP artifacts, compares them against the Pi incumbent on one review arena, and emits a report suitable for a G2-style decision.

Non-Goals

  • No production PR posting from Daedalus.
  • No generic arbitrary-lab framework before the Cerberus lab works.
  • No hardcoded reviewer roster as the product claim.
  • No self-approval of G1/G2/G3/G4/G5.
  • No weakening the real-repo freeze gate to get a result.

Verification System

Claim: Daedalus can compare autonomous review substrates for Cerberus credibly.

Falsifier: artifacts are not comparable, OpenCode output is not deterministic enough to score, or Daedalus cannot distinguish substrate failure from model quality.

Driver: Cerberus fixture run, then live OpenCode/OMP run, then Daedalus scoring/report generation.

Grader: artifact schema validation, score against hidden answer key, cost/latency lifecycle fields present, non-inconclusive freeze.

Evidence packet: run directory, Cerberus transcript/artifact, Daedalus summary, and substrate comparison report.

Cadence: fixture before live spend; gate before landing; fresh critic before recommendation.
