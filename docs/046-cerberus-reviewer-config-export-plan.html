Backlog 046 / Cerberus Handoff

Export the packet Cerberus actually imports.

Target outcome: a measured Daedalus review delivery writes a sandbox-only ReviewerConfigPacket.v1 JSON artifact that Cerberus validates, dry-run imports, and rejects for production promotion while G3/G4/G5 remain pending.

Chosen design: add one thin exporter target beside the existing delivery export. Reuse committed delivery, run, contract, approval, and prompt evidence. Do not build a generic promotion engine.

Quality System

Standards: Rust-only, evidence-preserving, deterministic config hash, no self-approval, no default mutation.

Proof: Daedalus unit tests, Daedalus CLI smoke, Cerberus validator/importer QA, full Daedalus gate, fresh critic.

Stop: if a field requires invented evidence, a pending gate is promoted, or Cerberus validation cannot be the oracle.

Source Evidence

deliveries/pr-review/agent.toml, contract.toml, runs/.../summary.json, trials.jsonl, and G2/G3 approval files are the source of truth.

Packet Shape

Emit packet_id, producer, benchmark, promotion gates, rollback, cost, harness/model metadata, prompt hashes, config hash, and embedded ReviewConfig.v1.

Cross-Repo Oracle

The exporter is not done until the generated JSON passes cerberus-cli validate-reviewer-config and import-reviewer-config --dry-run.
